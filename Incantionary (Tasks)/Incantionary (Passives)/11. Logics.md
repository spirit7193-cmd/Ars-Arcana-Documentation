This is where Akashic Magic becomes complex. Logic Incants allow a magus to make certain decisions within their spells based on various conditions (usually through the use of Query Incants). They are invaluable assets in spell theory and should never be lightly ignored. In this section, the Logics that the Akashic listens to may be found and studied.

**IF**

**SYNTAX**

_Akashic -> Boolean, List, List_

_If requires 3 Clusters to function properly._

_A Boolean to determine which List is chosen to be return._

_A List to represent the first potential option._

_Another List to represent the second potential option._

_Akashic <- List_

_If is the incant that represents binary decision. It considers the Boolean provided, and if it is true, the first List is returned. If it is false, the second List is returned. Whatever List is returned is added to the top of the Akashic Stack._

**Example Spells:**

ETE GELAQAR? ABI ETE GEA FUS VIR ABI ETE BIEGEA DEBUG VIR IF SINFAN -> If the caster is in water, conjures a light at their feet. If they are NOT in water, allows them to view the Akashic, which will reveal the position of their head.

NOTE: IF only returns a List. The reason why the Lists are cast is because of SINFAN.

**WHEN**

**SYNTAX**

_Akashic -> Boolean, List_

_When requires 2 Clusters to function properly._

_A Boolean to determine whether the List is returned._

_A List to be returned if the Boolean is True._

_Akashic <- List/Null_

_If is an incant that represents binary decision, much like IF but more concise. However, if the Boolean is true, the first List is returned; otherwise, Null is returned. Whatever is returned is added to the top of the Akashic Stack._

**Example Spells:**

ETE GELAQAR? ABI ETE GEA FUS VIR WHEN SINFAN -> If the caster is in water, conjures a light at their feet. Nothing happens otherwise.

NOTE: WHEN only returns a List or Null. The reason why the List is cast is because of SINFAN.

**OR**

**SYNTAX**

_Akashic -> Boolean, Boolean_

_Or requires 2 Boolean Clusters to function properly._

_Akashic <- Boolean_

_Or is the incant that represents "either/or". It determines whether one or both Booleans provided are true. If that is so, it returns a true Boolean. If neither of them are, a false Boolean is returned. Whatever is returned is added to the top of the Akashic Stack._

**Example Spells:**

ETE GELAQAR? ETE GELACOR? OR ABI ETE GEA FUS VIR WHEN -> If the caster is in water OR on the ground, conjures a magical light at their feet. Nothing happens otherwise.

**AND**

**SYNTAX**

_Akashic -> Boolean, Boolean_

_And requires 2 Boolean Clusters to function properly._

_Akashic <- Boolean_

_And is the incant that represents "and". It determines whether both Booleans provided are true. If that is so, it returns a true Boolean. If one or both of them are NOT true, a false Boolean is returned. Whatever is returned is added to the top of the Akashic Stack._

**Example Spells:**

ETE GELAQAR? ETE GELACOR? AND ABI ETE GEA FUS VIR WHEN -> If the caster is in water AND on the ground, conjures a magical light at their feet. Nothing happens otherwise.

**NOT**

**SYNTAX**

_Akashic -> Boolean_

_Not requires 1 Boolean Cluster to function properly._

_Akashic <- Boolean_

_Not is the incant that represents "not", however, it is one of the more transformative Logic Incants. It takes the Boolean provide and returns the opposite. If provided with true, it returns false. If provided with false, it will return true. Whatever is returned is added to the top of the Akashic Stack._

**Example Spells:**

ETE GELAQAR? NOT ETE GELACOR? AND ABI ETE GEA FUS VIR WHEN -> If the caster is NOT in water AND is on the ground, conjures a magical light at their feet. Nothing happens otherwise.

Note: This example makes use of two Logic Incants. This is because NOT does not compare; it transforms. AND, however, does, and so it can be used to ensure that the two booleans are compared.

**JIT**

**SYNTAX**

_Akashic -> Any, Any_

_Jit requires 2 Clusters to test for equality with._

_Akashic <- Boolean_

_Jit is the incant that represents "value equality". What this means is that if the information present in both Clusters are the same (regardless of type), it will return a true Boolean. Otherwise, a false Boolean is returned. Whatever is returned is added to the top of the Akashic Stack._

**IROJIT**

**SYNTAX**

_Akashic -> Any, Any_

_Irojit requires 2 Clusters to test for equality with._

_Akashic <- Boolean_

_Irojit is the incant that represents "false equality". What this means is that if the types of both Clusters are the same, it will return a true Boolean. Otherwise, a false Boolean is returned. Whatever is returned is added to the top of the Akashic Stack._

_Note that this means that two Entity Clusters, two Block Clusters, two Vector Clusters, etc. will always show that they are equal under IROJIT._

**IRAJIT**

**SYNTAX**

_Akashic -> Any, Any_

_Irajit requires 2 Clusters to test for equality with._

_Akashic <- Boolean_

_Irajit is the incant that represents "true equality". What this means is that if the information present in both Clusters are the same in both type and value, it will return a true Boolean. Otherwise, a false Boolean is returned. Whatever is returned is added to the top of the Akashic Stack._

**JIRVAR**

**SYNTAX**

_Akashic -> Number, Number_

_Jirvar requires 2 Number Clusters to compare._

_Akashic <- Boolean_

_Jirvar is the incant that represents "less than". If the first numberis less than the second number, it returns a true Boolean. Otherwise, a false Boolean is returned. Whatever is returned is added to the top of the Akashic Stack._

**JIRVARJIT**

**SYNTAX**

_Akashic -> Number, Number_

_Jirvarjit requires 2 Number Clusters to compare._

_Akashic <- Boolean_

_Jirvarjit is the incant that represents "less than or equal to". If the first number is less than OR equal to the second number, it returns a true Boolean. Otherwise, a false Boolean is returned. Whatever is returned is added to the top of the Akashic Stack._
