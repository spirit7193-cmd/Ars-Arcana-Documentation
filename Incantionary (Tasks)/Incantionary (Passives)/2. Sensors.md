An important requirement of most spells is the ability to sense the mob, item or block in reality you want to affect. In this section, there are incants that can translate physical things into their akashic variants; variants a magus can use.

**ESIJIT**

**SYNTAX**

_Akashic -> Entity_

_Esijit requires an entity. This entity will serve as the eyes of the incant._

_Akashic <- Entity/Null_

_Fananim returns a Number Cluster, representing the amount of arcana a target has._

_ESIJIT is the incant that represents "seeing others". It gathers the entity the observer is directly looking at. Naturally, this means that spells created with ESIJIT require some amount of ocular precision. If nothing is detected, a Null Cluster is returned._

**Example Spells:**

ETE ESIJIT DEBUG -> Allows the caster to see the Akashic and view the Cluster returned, which will either be an Entity or Null.

ETE ESIJIT GEA FUS -> Detects the entity the caster is looking at and conjures a magical light at their feet.

**ESIFIR**

**SYNTAX**

_Akashic -> Entity_

_Esifir requires an entity. This entity will serve as the eyes of the incant._

_Akashic <- Block/Null_

_ESIFIR is the incant that represents "seeing blocks". It gathers the block that the observer is directly looking at. If nothing is detected, a Null Cluster is returned._

**Example Spells:**

ETE ESIFIR DEBUG -> Allows the caster to see the Akashic and view the Cluster returned, which will either be a Block or Null.

ETE ESIFIR GEA HUSCOR -> Detects the block the caster is looking at and shatters it (if it is mineable).

**FIRUGEA**

**SYNTAX**

_Akashic -> Vector_

_Firugea requires a vector, which will serve as a position._

_Akashic <- Block_

_Firugea is the incant that represents "find block". It gathers the block at the position provided if that area is loaded. If the position is not loaded, the entire spell will error._

**Example Spells:**

ETE GEA FIRUGEA DEBUG -> Allows the caster to see the Akashic and view the block at their feet. Note that this is NOT the block they are standing on and this spell will likely return "minecraft:air".

**ENIJIT**

**SYNTAX**

_Akashic -> Vector, Vector_

_Enijit requires two vectors. The first one describes the position that the Incant's effects will center around. The second one describes the direction the incant will actually look in._

_Akashic <- Entity/Null_

_Enijit is the incant that represents "to see others remotely". It is a variation of ESIJIT, but the entity gathered does not directly have anything to do with the caster's line of sight._

_Instead, the sight line can target things in and around any_

_position, so long as it is loaded and the entity is in line with the direction provided._

_Like ESIJIT, if no entity is found, Null is returned._

**Example Spells:**

ETE BIEGEA ETE ESILIB ENIJIT -> From the head position of the caster, this spell fires an invisible ray in the same direction they are looking. Any entity that intercepts this ray is added to the Akashic.

**ENIFIR**

**SYNTAX**

_Akashic -> Vector, Vector_

_Enifir requires two vectors. The first one describes the position that the Incant's effects will center around. The second one describes the direction the incant will actually look in._

_Akashic <- Block/Null_

_Enifir is the incant that represents "to see a block remotely". It is a variation of Esifir, but the entity gathered does not directly have anything to do with the caster's line of sight._

_Instead, the sight line can target things in and around any_

_position, so long as it is loaded and the block detected is in line with the direction provided._

_Like Esifir, if no block is found, Null is returned._

**Example Spells:**

ETE BIEGEA ETE ESILIB ENIFIR -> From the head position of the caster, this spell fires an invisible ray in the same direction they are looking. Any block that intercepts this ray is added to the Akashic.

**ESIZON**

**SYNTAX**

_Akashic -> Entity, Number, Number, Number_

_Esizon is a complex Incant that requires 4 Clusters. In order, an Entity Cluster is needed to serve as the "observer" of the Incant_

_A Number Cluster is necessary to determine how wide (in degrees) ESIZON's detection cone is._

_Another Number Cluster is necessary to determine how long the cone is. Eg, if the cone goes on for 10 blocks._

_Finally, the last Number Cluster serves as a numeric filter that ranges from 0-10. This controls which type of entity is seen and which type is ignored. A key is provided below\._

_Akashic <- List (Entity)_

_Esizon is the incant that represents "others in field of vision". It is another variation of ESIJIT, but it collects more than 1 Entity and places them in a special Cluster Type, a List._

_Spells using ESIZON usually contain YUTSINFAN (primarily because of the List ESIZON returns). However, this is considered an advanced Incant that functions entirely within the realm of the Akashic. Arcanic beginners aren't primarily seen using ESIZON or YUTSINFAN early on for this reason._

**Example Spells:**

ETE 15 10 0 ESIZON DEBUG -> Allows the caster to see the Akashic and view the Clusters returned. In this case, all entities 10 blocks in front of the caster within a 15° visual field. Quite a confusing description, I'm sure.

**Numeric Filter Key**

0 = All Entities

1 = Only Players

2 = All except Players

3 = Only Monsters

4 = All except Monsters

5 = Only Projectiles

6 = All except Projectiles

7 = Only Items

8 = All except Items

9 = Only Magi

10 = All except Magi

**WIX**

**SYNTAX**

_Akashic -> Vector, Number, Number_

_Wix is an Incant that requires 3 Clusters. In order, a Vector Cluster that represents the location that the Incant will detect around._

_A Number Cluster is necessary to determine how big (in blocks) WIX's circle detection is.,_

_Finally, the last Number Cluster serves as a numeric filter§r that ranges from 0-10. A key is provided below\._

_Akashic <- List (Entity)_

_WIX is the incant that represents "entities in a bubble radius". It, like ESIZON, collects more than 1 Entity and places them in a List. WIX requires YUTSINFAN to actually affect anything it detects. However, that is considered advanced akashic manipulation so we will not delve further this early._

**Example Spells:**

ETE GEA 10 0 WIX DEBUG -> Allows the caster to see the Akashic and view the Clusters returned. In this case, all entities in a 10 block radius of the caster's feet.
