There are times when magi have questions that need answers. There are also times when the Akashic can answer. In this section, the Incants that ask will be revealed. They are considered Query Incants and they are easily identified by the '?' that end each and every one of them.

**MALEME?**

**SYNTAX**

_Akashic -> Entity_

_Maleme? requires an entity to function._

_Akashic <- Boolean_

_Maleme? is the query that asks "are they running?". It checks if the entity is running and returns a Boolean, which is added to the top of the Akashic Stack. If they are, true is returned. If they aren't, false is returned._

**Example Spells:**

ETE MALEME? DEBUG -> Allows the caster to see the Akashic, which will tell them whether or not they are running.

**EXIRER?**

**SYNTAX**

_Akashic -> No Clusters_

_Exirer? doesn't require any Clusters. What is being asked relates to the spell itself._

_Akashic <- Number_

_Exirer? is the query that asks "how many loops have passed?". A "loop" is what each instance of an Ouroboros spell is called, beginning at '0'. This number is added to the top of the Akashic Stack._

**Example Spells:**

EXIRER? DEBUG -> Allows the caster to see the Akashic, showing them their current loop. As this is not an Ouroboros spell, the number will be 0.

**TULENI?**

**SYNTAX**

_Akashic -> Entity/Block/Frame Item/Item_

_Tuleni? requires either an entity, a block or an item._

_Akashic <- Word_

_Tuleni? is the query that asks "what is this entity/block/item?". It is answered by giving the true name of the thing given. This Word Cluster is added to the top of the Akashic Stack._

**Example Spells:**

ETE TULENI? DEBUG -> Allows the caster to see the Akashic, showing them their type id. In the case of a player, "minecraft:player" is what will be detected.

**TULNIM?**

**SYNTAX**

_Akashic -> Frame Item/Item_

_Tulnim? requires an item to function._

_Akashic <- Number_

_Tulnim? is the query that asks "how many of this item is there?". It is answered by giving the quantity of the item in the provided Item Cluster. This Number Cluster is added to the top of the Akashic Stack._

**Example Spells:**

ETE 0 BURITE TULNIM? DEBUG -> Allows the caster to see the Akashic, showing the amount of the item in the first hotbar slot they have. If nothing is there, an error will be triggered.

**ITESULNIM?**

**SYNTAX**

_Akashic -> Entity_

_Itesulnim? requires an entity to function._

_Akashic <- Number_

_Itesulnim? is the query that asks "which slot do they have selected?". This naturally means that the entity must be a player, as only we have this ability. The index number found is added to the top of the Akashic Stack._

**Example Spells:**

ETE ITESULNIM? DEBUG -> Allows the caster to see the Akashic, showing the slot they have selected. Keep in mind that, like Lists, counting slots begins at 0, not 1.
