In arcanic, there are incants that represent things that do not change. They are constant, never changing. As a result, these incants are always reliable in the return of their values. In this section, these incants will be highlighted, regardless of how useful (and the entire opposite) they are.

**HET**

**SYNTAX**

_Akashic -> No Clusters_

_Het does not require clusters to function. You'll find that Constants typically don't._

_Akashic <- Null_

_Het is the incant that signifies "nothing, void, things unknown". Although Null tends to indicate that there is nothing, any studied Magus understands that Null is all that which cannot be understood completely by our craft, likely to protect us to some degree. This Null Cluster is added to the top of the Akashic Stack._

**Example Spells:**

HET DEBUG -> Allows the caster to see the Akashic and view a Null Cluster.

**MALIBHET**

**SYNTAX**

_Akashic -> No Clusters_

_Malibhet does not require clusters to function._

_Akashic <- Vector_

_Malibhet is the incant that signifies "nowhere". It is as metaphorical as it is physical. The Vector returned is \[0, 0, 0], which represents Origin, implying that Nowhere and Origin are equivalent. Whether this is true, it has not been proven as a fact. The Vector Cluster is added to the top of the Akashic Stack._

**Example Spells:**

MALIBHET DEBUG -> Allows the caster to see the Akashic and view a Vector Cluster, which will always be \[0, 0, 0].

**SYNTAX**

_Akashic -> No Clusters_

_Hafozon does not require clusters to function._

_Akashic <- Number_
