Akashicals affect the Akashic itself, manipulating it according to the Magus's intent. These Incants truly manipulate information and the Akashic itself, so they have been given their own section.

**XABAS**

**SYNTAX**

_Akashic -> Vector_

_Xabas requires 2 Clusters to function at all._

_Akashic <- No Cluster_

_Xabas is the incant that represents a "swapping". The two Clusters taken have their place in the Akashic swapped. As a result, nothing is added to or removed from the Akashic Stack._

**SINJIT**

**SYNTAX**

_Akashic -> Any_

_Sinjit requires a Cluster to function at all._

_Akashic <- Any_

_Sinjit is the incant that represents a "duplication". The Cluster provided is duplicated and returned. The duplicated Cluster is added to the top of the Akashic Stack._

**YINGIM**

**SYNTAX**

_Akashic -> Any_

_Yingim requires a Cluster to function at all._

_Akashic <- No Clusters_

_Yingim is the incant that represents an "erasure". The Cluster provided is destroyed. No Cluster is re-added to the Akashic because of this._

**XIBFAN**

**SYNTAX**

_Akashic -> Number_

_Xibfan requires a Number Cluster to function._

_Akashic <- No Clusters_

_Xibfan is the incant that represents a "delay". The Akashic freezes. All magic stops for as long as the number dictates (which may not be more than 30 seconds). After this delay, the spell will continue as normal, if it can._

**HUSGIM**

**SYNTAX**

_Akashic -> No Clusters_

_Husgim does not require any clusters to function._

_Akashic <- No Clusters_

_Husgim is the incant that represents a "breakage". The Akashic cracks and the spell that encounters it shatters. It fizzles into oblivion, leaving nothing for anything to return to._
