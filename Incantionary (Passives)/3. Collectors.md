Certain incants are designed to extract further information from Entity and Block Clusters. In this section, we will look over the most basic of these.

**GEA**

**SYNTAX**

_Akashic -> Entity/Block/Node_

_Gea requires either an entity, a block or a node to function._

_Akashic <- Vector_

_Gea is the incant that signifies "position". In the case of blocks and nodes, it gathers their position in the world. In the case of entities, the position of their feet is gathered instead. The Vector gathered is added to the top of the Akashic Stack._

**Example Spells:**

ETE GEA DEBUG -> Allows the caster to see the Akashic and view their position.

ETE GEA FUS -> Conjures a magical light at the caster's feet.

**BIEGEA**

**SYNTAX**

_Akashic -> Entity_

_Biegea requires an entity to function._

_Akashic <- Vector_

_Biegea is the incant that signifies "head position". Unlike GEA, it only functions off entities, as only they have head positions. The Vector gathered is added to the top of the Akashic Stack._

**Example Spells:**

ETE BIEGEA DEBUG -> Allows the caster to see the Akashic and view their head position.

ETE BIEGEA FUS -> Conjures a magical light at the caster's head.

**ESILIB**

**SYNTAX**

_Akashic -> Entity_

_Esilib requires an entity to function._

_Akashic <- Vector_

_Esilib is the incant that signifies "view direction" or "where they are looking". Instead of a position, the Vector Cluster returned represents a direction; the direction the entity is looking in. Thus Vector is then added to the top of the Akashic Stack._

**Example Spells:**

ETE ESILIB DEBUG -> Allows the caster to see the Akashic and view the Vector representing the direction they're looking in.

**ESIPAL**

**SYNTAX**

_Akashic -> Entity_

_Esipal requires an entity to function._

_Akashic <- Vector_

_Esipal is the incant that signifies "see block face". This might be a confusing one. It directly gathers the face of the block the entity is looking at and returns the direction Vector they are looking at it from. That is, if the entity is looking at the North face of a block, it will return the direction representing South. This Vector is added to the top of the Akashic Stack._

**Example Spells:**

ETE ESIFIR GEA ETE ESIPAL SINVERA FUS -> In order:

        - Gathers the block the caster is looking at with ETE ESIFIR.

        - Gathers the block's position in the world using GEA.

        - Recalls the caster (using ETE), and uses that Entity Cluster to use ESIPAL, which gathers the face of the block detected.

        - SINVERA adds together the block's position and the block's face. This will create a Vector position where there is likely no block, just air.

        - FUS uses this new Vector position to create a brilliant, magical light at this empty position!

**ESIPALGEA**

**SYNTAX**

_Akashic -> Entity_

_Esipalgea requires an entity to function._

_Akashic <- Vector_

_Esipalgea is the incant that signifies "see accurate block face position". This might also be another confusing one. It will directly take the block seen and its face, and return exactly where on that face the entity is looking, as a position around the Vector \[0, 0, 0]. This new Vector is added to the top of the Akashic Stack._

**Example Spells:**

ETE ESIPALGEA DEBUG -> Allows the caster to see the Akashic and view the position on the face of the block they are looking at.

**IRAPAL**

**SYNTAX**

_Akashic -> Entity, Vector_

_Irapal requires an entity, who will serve as the point of origin, and a vector direction, that will be grafted to the entity, to function._

_Akashic <- Vector_

_Irapal is the incant that signifies "relative direction". By taking the entity and a provided vector direction, IRAPAL uses the entity's view direction to find the relative direction that is equivalent to the provided vector direction. This new Vector is added to the top of the Akashic Stack._

**Example Spells:**

ETE MALIBDIE IRAPAL DEBUG -> Allows the caster to see the Akashic and view the vector direction representing the caster's East. Note that MALIBDIE is the Constant Incant representing "East". To get a better feel of IRAPAL, any of the Direction Constants can be substituted with MALIBDIE.
