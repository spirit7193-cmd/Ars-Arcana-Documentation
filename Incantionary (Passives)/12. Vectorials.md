Akashic Vectors are positions. Akashic Vectors are directions. Akashic Vectors are Lists of 3 numbers. Magi have many ideas of what exactly these Clusters are, most of which are correct. In this section, there are Incants that manipulate this necessary Cluster Type.

**SINAFIL**

**SYNTAX**

_Akashic -> Number, Number, Number_

_Sinafil requires 3 Number Clusters to function._

_Akashic <- Vector_

_Sinafil is the incant that is taken to mean "create Vector from Numbers". It considers the numbers provided and turns them into a Vector (x, y, and z). The Vector is added to the top of the Akashic Stack. It is considered powerful for teleportation spells._

**Example Spells:**

0 0 0 SINAFIL DEBUG -> Changes 0, 0, and 0 into a Vector Cluster, which is viewed by the caster through DEBUG.

**YINAFIL**

**SYNTAX**

_Akashic -> Vector_

_Yinafil requires 1 Vector Cluster to function._

_Akashic <- Number, Number, Number_

_Yinafil is the incant that is taken to mean "break Vector into Numbers". It considers the vector provided and turns it into a series of numbers. Each coordinate is added into the Akashic in order, so by the end, the third number will be at the top of the Akashic Stack._

**Example Spells:**

ETE GEA YINAFIL DEBUG -> Changes the caster's position into a series of Number Clusters, which are viewed by the caster through DEBUG.

**LIBAFIL**

**SYNTAX**

_Akashic -> Vector_

_Libafil requires 1 Vector Cluster to function._

_Akashic <- Number_

_Libafil is the incant that is taken to mean "vector length". It considers the vector provided and returns a number representing how long it is. This number is added to the top of the Akashic Stack._

**Example Spells:**

ETE ESILIB 2 TOCOVER LIBAFIL DEBUG -> Gets the caster's view vector and multiplies it by 2. Its length is then deduced, which is viewed by the caster through DEBUG as a Number Cluster.

**LIBAS**

**SYNTAX**

_Akashic -> Vector, Vector_

_Libas requires 2 Vector Clusters to function._

_Akashic <- Number_

_Libas is the incant that is taken to mean "distance between vectors". As Vectors can signify positions, this Incant gets the distance between two Vectors (in blocks) and returns it as a number. This number is added the top of the Akashic Stack._

**Example Spells:**

0 0 0 SINAFIL ETE GEA LIBAS DEBUG -> Gets the distance between 0,0,0 and the caster's position, which is then viewed by the caster through DEBUG.

**IROKIE**

**SYNTAX**

_Akashic -> Vector_

_Irokie requires 1 Vector Cluster to function._

_Akashic <- Vector_

_Irokie is the incant that has no particular meaning. It rounds off each value in a Vector and returns it. This new Vector is added to the top of the Akashic Stack. There are other ways to do this of course, but this is the easiest._

**Example Spells:**

0.45 0.89 0.03 SINAFIL IROKIE DEBUG -> Gets the Vector created by SINAFIL and returns a new Vector (\[0, 1, 0]), which is then viewed by the caster through DEBUG.
